# UI Design & Styling Rules

このルールは、フロントエンドのコード（`.astro`, `.tsx`, `.css`）を生成・編集する際に**必ず**適用されます。
公式の `frontend-design` スキルと併用し、洗練されたベースデザインの上に「圧倒的な遊び心」と「独自性」を上乗せすることが目的です。

## 1. 遊び心 (Playfulness) と アートディレクション
- **インスピレーション:** 「Splatoon」のようなインクの飛び散り、アシンメトリー（非対称）、ネオ・ブルータリズム、ゲームUIの躍動感。
- **カラーパレット:** ダークテーマ（例: `#0F172A`）をベースに、鮮やかで蛍光的なアクセントカラー（ネオンイエロー、ネオンピンク、シアンなど）を大胆に使用する。
- **タイポグラフィ:** 見出しには個性的で太いフォントを使用し、サイズは極端に大きく（例: `8rem` 以上）してインパクトを出す。文字の角度を少し傾けるなどの工夫も推奨。

## 2. 驚き (Wow Factor) と アニメーション
- **GSAPの必須化:** 静的な要素を極力減らす。要素の登場、ホバー、スクロール時には、必ずGSAP（ScrollTrigger等を含む）を用いて、粘り気のある、または弾けるようなアニメーションを付与する。
- **予測不能なレイアウト:** 均等なグリッドに縛られず、要素をあえて重ねる（`z-index`の活用）、斜めに配置する（`transform: rotate`）、画面全体をキャンバスとして使うなど、視覚的な驚きを作る。

## 3. 厳格な禁止事項 (Negative Prompts)
以下の要素やスタイルを使用することは**厳禁**です。
- **Tailwind CSSの禁止:** ユーティリティクラス（`bg-blue-500`, `p-4`, `flex` など）は絶対に使用しない。スタイリングはすべて CSS Modules を使用すること。
- **量産型UIの禁止:** 白背景、均等な角丸、薄いドロップシャドウで構成された「よくあるSaaS風のカードUI」。
- **退屈なインタラクションの禁止:** `opacity` が変わるだけ、色が少し薄くなるだけの「無難なホバーアニメーション」。必ず動き（Scale, Bounce, Rotateなど）を伴うこと。

## 4. コーディングと命名規則
- CSS Modulesのクラス名は、役割が明確で世界観に沿った名前にする（例: `splatButton`, `neonHeading`, `inkDropWrapper`）。
- Astroコンポーネント内では `<style>` タグ、Reactコンポーネント内では `*.module.css` ファイルを使用すること。
