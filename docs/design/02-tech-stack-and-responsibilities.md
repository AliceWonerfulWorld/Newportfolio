# 2. 技術スタックと責務の定義 (Tech Stack & Responsibilities)

本プロジェクトでは、「遊び心（Wow Factor）」と「高いパフォーマンス」を両立させるため、以下の技術スタックを採用し、それぞれの責務を厳格に分離する。

## 2.1 コアフレームワーク: Astro
* **役割 (責務):** サイト全体の基盤、静的ページの生成（SSG）、ルーティング、レイアウトの構築。
* **採用理由:**
    * ポートフォリオやブログなど、コンテンツ主体のWebサイトにおいて最速の表示パフォーマンス（Zero JavaScript by default）を実現できるため。
    * 動的なコンポーネントを必要な箇所にだけマウントできる「Astro Islands」アーキテクチャが、本プロジェクトの要件に完全に合致しているため。
* **使用箇所:** `src/pages/`, `src/layouts/`, `src/components/astro/` (静的なUIパーツ)

## 2.2 UIコンポーネント (動的): React
* **役割 (責務):** ユーザーのインタラクション（クリック、マウス追従）によって状態（State）が複雑に変化するコンポーネントの構築。
* **採用理由:**
    * 複雑な状態管理や、外部ライブラリ（キャンバス描画、ミニゲームの実装など）との相性が良いため。
    * 過去の成果物（TypeDefenseなどのReactベースのアプリ）を、記事やポートフォリオ内に直接マウント（埋め込み）しやすくするため。
* **使用箇所:** `src/components/islands/` 内に限定（静的なガワにはReactを使用しない）。

## 2.3 スタイリング: CSS Modules (プレーンCSS)
* **役割 (責務):** 各コンポーネントのレイアウト、タイポグラフィ、カラーリングなどの視覚表現（カプセル化）。
* **採用理由 (Tailwind CSSの禁止):**
    * AIによるコーディングで発生しがちな「量産型UI（ユーティリティクラスの組み合わせによる無難なデザイン）」を強制的に排除するため。
    * アシンメトリー（非対称）、要素の重なり（Z-index）、クリップパス（Clip-path）を用いた複雑な図形など、ドクター独自の世界観（Splatoon風、ネオンカラー）を精密に表現するには、コンポーネントごとに独立したプレーンなCSSを書く方が適しているため。
* **使用箇所:** `*.module.css` および Astroファイル内の `<style>` タグ。

## 2.4 アニメーション: GSAP (GreenSock Animation Platform)
* **役割 (責務):** スクロール連動アニメーション、ホバー時の弾力のある動き、DOM要素の複雑なタイムライン制御。
* **採用理由:**
    * 単純なCSSトランジションでは表現できない、インクが飛び散るような粘り気のあるモーションや、ゲーム的な躍動感を実装する「Wow Factor」の要となるため。
    * `ScrollTrigger` プラグインを使用し、ユーザーのスクロールに合わせて要素を歪ませたり、順番に出現させたりする高度なストーリーテリングを行うため。
* **使用箇所:** 主にクライアントサイドのスクリプト（Astroの `<script>` 内や、Reactの `useEffect` 内）。共通ロジックは `src/utils/animations.ts` に集約。

## 2.5 コンテンツ管理: Astro Content Collections (MDX + Zod)
* **役割 (責務):** 記事（ブログ）および作品データの管理、Markdownパーサー、Frontmatterの型定義。
* **採用理由:**
    * 外部のヘッドレスCMSを導入する複雑さを避け、コードと同じGitリポジトリ内で完結するMarkdownベースの執筆体験（DX）を得るため。
    * `MDX` をサポートしており、記事の文章中にReactコンポーネント（動的な図解やミニゲーム）を直接配置できるため。
    * `Zod` を用いたスキーマ定義により、データの記述漏れや型エラーをビルド時に確実に検知できるため。
* **使用箇所:** `src/content/` 配下。

## 2.6 ホスティング・デプロイ (予定)
* **プラットフォーム:** Vercel または Cloudflare Pages
* **役割:** GitHubリポジトリの `main` ブランチへのPushをトリガーとした、自動ビルドおよびグローバルエッジネットワークへの配信。
