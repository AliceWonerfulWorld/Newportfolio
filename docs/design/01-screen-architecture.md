# 画面・コンテンツ構成設計 (Screen & Content Architecture)

## 1. 全体コンセプト
シームレスな体験を提供する「シングルページ（1ページ）構成」をトップページのベースとし、作品や記事の詳細については動的ルーティングで個別ページへ遷移するハイブリッド型を採用する。
単なる情報の羅列ではなく、GSAPを用いたインクの飛び散りやスクロール連動アニメーションを駆使し、ゲームのステージを進むような没入感と「遊び心」を体現する。

## 2. ページ構成と主要コンテンツ

### 2.1 トップページ (`/`)
サイトの顔であり、ドクターの世界観と技術力を一目で伝えるメインキャンバス。

* **セクション1: Hero (トップビュー)**
    * **コンテンツ:** 画面全体を使った巨大なタイポグラフィ（「DOCTOR」または独自の世界観を表すキャッチコピー）。
    * **UI/UX:** マウスカーソルや初期スクロールに反応して、背景にカラフルなインク（ネオンカラー）が弾け飛ぶダイナミックなインタラクション（React + GSAP）。

* **セクション2: Profile & Vision (自己紹介と展望)**
    * **コンテンツ:** * 福岡工業大学での学びの集大成と、ウイングアーク1stでの開発エンジニアとしてのキャリアに向けたビジョン。
        * C、Java、Pythonなど、これまでのプログラミング経験のバックグラウンド。
        * 情報技術倫理に対する視点や、テクノロジーと社会の関わりについての考え。
    * **UI/UX:** テキストが単調に並ぶのを避け、スクロールに合わせてキーワードがハイライトされたり、アシンメトリーな配置でテキストが浮き上がってくる演出。

* **セクション3: Selected Works (代表作のショーケース)**
    * **コンテンツ:** 以下のような代表的な開発実績のピックアップ。
        * Magic Wand (Wind Waveハッカソン特別賞作品)
        * 魔法少女変身 WebARアプリ
        * TypeDefense (タイピングゲーム)
        * サクサクトレカ (TCGアプリ)
    * **UI/UX:** 均等なグリッド配置（量産型カードUI）は厳禁。要素を重ねたり、斜めに配置したり、スクロールに連動して作品のサムネイルが飛び出してくる予測不能なレイアウト。

* **セクション4: Dev Logs / Articles (記事・開発記)**
    * **コンテンツ:** 最新の執筆記事（ハッカソン参加レポート、技術的な試行錯誤、考察など）のサマリーを数件表示。
    * **UI/UX:** リストではなく、マスキングテープや蛍光マーカーでラフに束ねたような、手作り感とポップさが同居する独自デザインのリンク要素。

* **セクション5: Contact & Links (フッター領域)**
    * **コンテンツ:** GitHub、Zenn/Qiita（あれば）、SNSへのリンク。
    * **UI/UX:** ホバー時にアイコンが跳ねる（Bounce/Elastic）ような、粘り気のある心地よいマイクロインタラクション。

### 2.2 作品詳細ページ (`/works/[slug]`)
MDXを利用し、各プロジェクトの深い魅力を伝えるページ。

* **コンテンツ:**
    * 作品タイトル、キャッチコピー、サムネイル画像（独自に切り抜かれたデザイン）。
    * 技術スタックタグ（独自デザインのバッジ）。
    * デモURL、GitHubリポジトリへのリンク。
    * 開発の裏話、アーキテクチャ図、工夫したポイントの解説。
* **UI/UX:** MDXの強みを活かし、必要に応じて記事の途中にReactコンポーネント（例：TypeDefenseのミニゲーム自体）を直接埋め込み、読みながら体験できる「Wow Factor」を実装する。

### 2.3 記事詳細ページ (`/blog/[slug]`)
技術的なアウトプットや考察を発信するページ。

* **コンテンツ:** マークダウンベースのテキスト本文、ソースコードのシンタックスハイライト。
* **UI/UX:** 読みやすさを確保しつつも、リンクの色や引用ブロックのデザインにネオンカラーのアクセントを入れ、サイト全体の世界観（Vibe）を損なわないスタイリングを行う。

### 2.4 404 Not Found (`/404`)
* **コンテンツ:** ページが見つからない旨のメッセージ。
* **UI/UX:** ただのエラー画面にせず、画面中がインクで塗りつぶされているなど、エラーさえも遊び心に変えるデザイン。

## 3. 拡張性と運用フロー
* 新しい作品や記事は、`src/content/works/` または `src/content/blog/` にMDXファイルを追加するだけで自動的にルーティングと一覧が生成される静的サイト生成（SSG）モデルとする。
* ZodによるFrontmatterの厳格な型定義を行い、運用時の記述漏れやビルドエラーを防ぐ。
