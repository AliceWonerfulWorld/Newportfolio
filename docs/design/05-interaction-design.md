# 5. インタラクション設計 (Interaction Design)

本プロジェクトの最優先事項である「遊び心」と「Wow Factor」を、GSAPを用いた動的な演出によって定義する。単なるアニメーションではなく、ユーザーの操作（スクロール、マウス移動）に呼応する「手触り感」のある体験を追求する。

## 5.1 グローバル・インタラクション

サイト全体で共通して適用される、世界観を構築するための演出。

* **カスタムカーソル (Custom Cursor):**
    * 標準のカーソルを隠し、ネオンカラーの「インクの滴」のような形状のカーソルをReact（islands）で実装。
    * 停止時は円形、移動時は進行方向へ少し伸びる（メタボール的な動き）演出を加える。
* **インク・トランジション:**
    * ページ遷移時やセクションの切り替わり時に、画面の端からインクが塗りつぶされるようなワイプ演出（GSAP Timeline）を挿入する。

## 5.2 セクション別演出 (Key Moments)

### 5.2.1 Hero: 爆発的な登場
* **Initial Loading:** ページ読み込み完了時、「DOCTOR」のタイポグラフィがバラバラのパーツから中央へ集まり、背後で巨大なインクのしぶきが弾ける（Splat Effect）。
* **Mouse Follow:** 背景のインクがマウスカーソルの動きに微妙に反応し、液体のような揺らぎを見せる（GSAP + CSS Filter: blur/contrast）。

### 5.2.2 Scroll-driven Reveal (スクロール連動)
* **不均等な出現:** 要素が下から上にフェードインするだけの演出は禁止。
* **パララックス & 回転:** スクロール量に合わせて、作品のカードや画像が異なる速度で移動し、わずかに回転（rotate）しながら浮き上がってくる演出。
* **Sticky Storytelling:** プロフィールセクションでは、背景のテキストが固定（Sticky）され、スクロールに合わせて色が塗りつぶされていくような「読み進める楽しさ」を作る。

## 5.3 マイクロ・インタラクション (手触り感)

ボタンやリンクなど、最小単位の操作に対するフィードバック。

* **弾力のあるホバー (Elastic Hover):**
    * ボタンをホバーした際、単に色が変わるのではなく、スライムのように「プニッ」と膨らむ、または跳ねるような動き（GSAPの `Elastic.easeOut`）を実装する。
* **インク・ハイライト:**
    * ナビゲーションメニューをホバーすると、文字の下に蛍光マーカーで線を引いたようなインクが素早く走る演出。
* **アクティブ・フィードバック:**
    * クリックした瞬間に、その地点から小さなインクのパーティクルが数粒飛び散る演出。

## 4. 実装ガイドライン (GSAP Strategy)

1. **イージングの統一:**
   - 基本的には `power2.out` または `expo.out` を使用し、キビキビとした動きにする。
   - 「遊び心」を出す箇所には `elastic.out(1, 0.3)` や `back.out(1.7)` を積極的に採用する。
2. **パフォーマンス配慮:**
   - `will-change: transform` を適切に使用し、GPU加速を有効にする。
   - スクロール連動（ScrollTrigger）は、要素が画面内にある時だけ動作するように最適化する。
3. **アクセシビリティ:**
   - `prefers-reduced-motion` メディアクエリを考慮し、ユーザーがアニメーションを抑制している場合は、過度な演出を控えつつ情報の視認性を確保する代替スタイルを提供する。
